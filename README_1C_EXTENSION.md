# Расширение 1С для MCP-агента

Этот каталог содержит BSL-код, который нужно перенести в расширение 1С, чтобы веб-агент мог получать реальные данные из любой конфигурации через ограниченный HTTP-сервис.

## Что перезаливать в расширение

Обязательные файлы:

| Файл | Куда вставлять в 1С | Назначение |
| --- | --- | --- |
| `UniversalMCPServer.bsl` | общий модуль `MCPUniversalServer` | универсальная серверная логика: ping, чтение метаданных, выбор разрешенного объекта, чтение строк |
| `UniversalMCPHTTPServiceModule.bsl` | модуль HTTP-сервиса | тонкий слой обработчиков URL, который вызывает общий модуль |

Опциональный файл:

| Файл | Куда вставлять в 1С | Назначение |
| --- | --- | --- |
| `UniversalMCPSettingsForm.bsl` | модуль формы обработки `НастройкаИсточниковMCP` | помогает выбрать объекты 1С и сформировать JSON источников для админ-панели веб-агента |

Старые файлы `EduAgentHTTPService.bsl` и `EduAgentUniversalDataService.bsl` оставлены только как учебные примеры. Для текущего универсального проекта их в расширение перезаливать не нужно.

## Общий модуль

Создай в расширении общий модуль:

```text
Имя: MCPUniversalServer
Сервер: Да
Вызов сервера: Да
Клиент: Нет
Глобальный: Нет
Экспорт: обычный общий модуль
```

Вставь в него содержимое `UniversalMCPServer.bsl`.

Экспортные процедуры и функции общего модуля:

| Функция | Кто вызывает | Что делает |
| --- | --- | --- |
| `Ping(ЗапросHTTP)` | HTTP-сервис | возвращает признак доступности, имя конфигурации и дату |
| `Metadata(ЗапросHTTP)` | HTTP-сервис | возвращает список доступных справочников, документов и регистров |
| `UniversalData(ЗапросHTTP)` | HTTP-сервис | читает ограниченную таблицу по выбранному объекту 1С |
| `Report(ЗапросHTTP)` | HTTP-сервис | совместимый обработчик, внутри вызывает `UniversalData` |
| `МетаданныеКонфигурации(...)` | форма настройки | возвращает метаданные объектов с полями и чувствительностью |

Важное ограничение: модуль не выполняет произвольный текст запроса от LLM. Агент передает только `datasetId`, роль, период, лимит строк и список разрешенных полей.

## HTTP-сервис

Можно использовать уже созданный HTTP-сервис, например `eduagent` с корневым URL `edu-agent`, или создать новый `MCPAgent` с корневым URL `mcp-agent`. Главное, чтобы URL в настройках веб-приложения совпадал с опубликованным адресом.

В HTTP-сервисе должны быть URL-шаблоны:

| Шаблон URL | Метод | Обработчик |
| --- | --- | --- |
| `ping` | `POST` | `Ping` |
| `metadata` | `POST` | `Metadata` |
| `universal-data` | `POST` | `UniversalData` |
| `report` | `POST` | `Report` |

В модуль HTTP-сервиса вставь `UniversalMCPHTTPServiceModule.bsl`.

Пример итогового адреса, если публикация базы называется `InfoBase4`, а корневой URL сервиса `edu-agent`:

```text
http://127.0.0.1:8081/InfoBase4/hs/edu-agent
```

Тогда проверки будут идти по адресам:

```text
POST /InfoBase4/hs/edu-agent/ping
POST /InfoBase4/hs/edu-agent/metadata
POST /InfoBase4/hs/edu-agent/universal-data
POST /InfoBase4/hs/edu-agent/report
```

## Проверка из PowerShell

```powershell
curl.exe -i -u "admin:ПАРОЛЬ" `
  -H "Content-Type: application/json" `
  -d "{}" `
  "http://127.0.0.1:8081/InfoBase4/hs/edu-agent/ping"
```

Метаданные:

```powershell
curl.exe -i -u "admin:ПАРОЛЬ" `
  -H "Content-Type: application/json" `
  -d "{""role"":""admin"",""includeFields"":true}" `
  "http://127.0.0.1:8081/InfoBase4/hs/edu-agent/metadata"
```

Чтение конкретного объекта:

```powershell
curl.exe -i -u "admin:ПАРОЛЬ" `
  -H "Content-Type: application/json" `
  -d "{""datasetId"":""1c://object/document/РекламационныеАкты"",""role"":""director"",""params"":{""periodStart"":""2026-01-01"",""periodEnd"":""2026-12-31"",""query"":""покажи рекламационные акты""},""limit"":50}" `
  "http://127.0.0.1:8081/InfoBase4/hs/edu-agent/universal-data"
```

Если имя документа другое, возьми точный `id` из ответа `/metadata`.

## Опциональная обработка настройки

Если нужна обработка для ручного выбора источников:

1. Создай обработку `НастройкаИсточниковMCP`.
2. Добавь форму.
3. Добавь реквизиты формы:

| Реквизит | Тип | Комментарий |
| --- | --- | --- |
| `ТаблицаОбъектов` | `ТаблицаЗначений` | таблица найденных объектов |
| `JSONИсточников` | `Строка`, многострочная | готовый JSON для админ-панели агента |
| `РолиПоУмолчанию` | `Строка` | например `admin,director` |

4. Вставь в модуль формы `UniversalMCPSettingsForm.bsl`.
5. На форме сделай команды:

| Команда | Обработчик |
| --- | --- |
| Обновить список объектов | `ОбновитьСписокОбъектов` |
| Сформировать JSON | `СформироватьJSONИсточников` |
| Скопировать JSON | `СкопироватьJSONВБуфер` |

Эта обработка не обязательна для работы HTTP-сервиса. Она нужна только для удобной подготовки списка источников и ролей.

## Что должно быть на стороне веб-клиента или другого MCP-клиента

Если вместо текущего сайта подключается другой веб-интерфейс, в нем нужно реализовать четыре операции:

| Операция клиента | HTTP-вызов 1С | Зачем нужна |
| --- | --- | --- |
| `ping_1c` | `POST /ping` | проверка доступности базы |
| `discover_1c_configuration` | `POST /metadata` | загрузка списка объектов и полей |
| `list_1c_objects` | данные из `/metadata` | показ источников пользователю или администратору |
| `query_1c_data` | `POST /universal-data` | получение табличных данных |

Для MCP-клиента эти операции можно опубликовать как tools:

```text
ping_1c
discover_1c_configuration
list_1c_objects
query_1c_data
```

Языковая модель не должна получать право выполнять произвольный запрос 1С. Она выбирает один из опубликованных источников, а 1С повторно проверяет роль, штатное право чтения и лимит строк.

## Роли и безопасность

Внутри `MCPUniversalServer` заложена базовая ролевая эвристика:

| Роль | Доступ |
| --- | --- |
| `admin` | все объекты, разрешенные штатными правами 1С |
| `director`, `rector` | все объекты, разрешенные штатными правами 1С |
| `accountant` | объекты с признаками бухгалтерии, банка, кассы, зарплаты, налогов, контрагентов |
| `study-office` | учебные объекты |
| `department-head` | объекты не выше средней чувствительности |

Для промышленной эксплуатации эту эвристику лучше заменить на регистр сведений или справочник настроек доступа в самой 1С: объект, роль, разрешенные поля, лимит строк, чувствительность.

## Что делать после переноса кода

1. Проверить расширение в конфигураторе.
2. Обновить конфигурацию базы данных.
3. Опубликовать HTTP-сервисы расширения на веб-сервере.
4. Проверить `/ping`.
5. Проверить `/metadata`.
6. Проверить `/universal-data` на одном конкретном объекте из `/metadata`.
7. В веб-приложении указать базовый URL 1С.
8. Нажать загрузку источников в администрировании и сохранить разрешенные источники.
